### ∥ Parallel Routes

> 병렬 라우트

- `@` 처리

```
archive/
├── @latest/
│   └── page.ts
├── @archive/
│   ├── page.ts
│   └── [slug]/
│       └── page.ts
└── layout.ts
```

- 자동으로 layout param으로 들어감 !

  ```js
  const ArchiveLayout({archive, layout}) => {
    return (
      <>
        {archive}
        {latest}
      </>
    )
  }
  ```

  - 문제: `/archive/1211` 불가 (latest > [slug]가 없음)
  - 해결: `latest/default.ts`
    - 병렬 라우팅 되더라도 얘 보여줌 (no matter which slug)
